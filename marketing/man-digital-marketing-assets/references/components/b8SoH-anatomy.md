# b8SoH Anatomy - Signal To Sales Play Map

Source: `/Users/romeoman/Documents/Marketing/Design/Pencil/Playground.pen`

Top node: `b8SoH`, 1190 x 1684.

## Composite Role

Use `b8SoH` when the graphic needs to explain how a signal becomes a sales play and then a concrete follow-up sequence inside HubSpot. The visual story is: a signal is detected, mapped to one of four play types, then routed through scoring, AI drafting, rep review, engagement, and measurement.

This is not a generic hub-and-spoke diagram. It is a signal-to-action operating map for sales plays.

## Central Hub

`OzxqD` is the central signal hub content. It contains a Medium Blue signal/wifi icon and the label `Signal Detected`. It sits over the circular hub surfaces `t9i2m6` and `RJSUv`.

Use this hub for the trigger point in a workflow. Keep it visually dominant; the play cards and action rows should feel downstream from it.

## Connector System

The connector system includes bracket groups `ENaum` and `DmmlV`, vertical line segments `SljRV`, `oXipF`, `S29uH`, `b8jx6`, and related path pieces. It visually links:

1. Signal hub to the play-card row.
2. Play cards to the action chain.
3. Action rows into a single vertical sequence.

If play-card count or action-row count changes, redraw connectors and screenshot the whole frame.

## Reuse Rules

- Clone `b8SoH` for a complete signal-to-sales-play map.
- Clone `OzxqD` plus the hub surfaces when a graphic needs a central signal trigger.
- Clone one of the four play cards only when the new graphic keeps the same card style.
- Clone the whole action chain when the process is score, draft, review, engage, measure.
- Use `tMsEe` instead when the story is about signal sources and provider inputs.
- Use `KVqAt` instead when the story is a five-stage data/AI/action architecture.
- Use `nRPmP` instead when the story is a circular flywheel.

## Coverage Boundary

This registry documents semantic reusable components: hub, play cards, action rows, connectors, title/logo, and decorations. It does not create standalone docs for every raw vector path inside icons, logo glyphs, rings, or dot grids.

## Composite Role

Use `llyux` when the graphic needs to explain a full operating model for turning signals into pipeline. The visual story is: capture high-quality buying signals, clean/enrich/score them, activate outreach, then measure and improve the engine.

This frame is best for implementation blueprints, maturity models, checklist-style operating models, and "what capabilities do we need?" articles.

## Reuse Rules

- Clone `llyux` for a complete blueprint or operating model.
- Clone a full layer row when explaining one layer in detail.
- Clone individual capability tiles only inside matching row/card styling.
- Keep layer labels vertical and numbered; they are part of the blueprint grammar.
- Keep bottom outcomes parallel in grammar.
- Use `KVqAt` for architecture flows, `tMsEe` for source-to-engine-to-outcome diagrams, and `llyux` for capability/model checklists.

## Coverage Boundary

This registry documents semantic reusable components: layer rows, capability tiles, outcome chain, connectors, title/logo, and decorations. It does not create standalone docs for every raw vector path inside icons, logo glyphs, line segments, or dot grids.

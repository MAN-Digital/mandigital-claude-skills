# Style Variants — Vary the Look

**Do not render every asset in the same solid-blue + orange + centered-title style.** The
Playground has a real range of cover treatments (Cover Variations A–E in 16:9 and 4:5, plus
white-card and light templates). Pick a variant that fits the message and channel, and **mix
variants across a set** so a campaign kit has visual range. The brand _system_ stays constant
(Montserrat/Lato, the real color variables, the logo, the accent palette); the _treatment_
varies (background, layout, eyebrow style, decoration).

Real palette variables (`get_variables` on `Playground.pen`): Primary Blue `#000FC4`,
Orange `#F26419`, Subtle Blue `#5963D9`, Ghost White `#F7F7FF`, MD Black `#434343`,
light `#C8CCF2` (subtitles). No cyan as a default.

## Treatment levers you can mix (independent of palette)

- **Layout:** centered hero vs left text column + right visual.
- **Eyebrow:** plain kicker (orange text) vs **pill chip** (variant 4) vs none.
- **Title accent:** none vs **left orange bar** (variant 5).
- **Hero visual:** orbit, lifecycle chain, white-card cluster, single large icon, or photo placeholder.
- **Decoration:** soft circles + glow, dot-grid, faint ring/radar, or none (dark spotlight).

## Rotation Rule (MANDATORY)

1. **Single asset:** choose the variant that fits the topic/channel — don't auto-default to
   Solid Blue. State which variant you picked and why.
2. **A set / campaign kit** (e.g. a webinar's post + ad + story + cover): **do not render them all
   in the same variant.** Mix at least a dark/blue variant with a light or gradient one so the set
   has range. Keep one consistent hero motif if useful, but vary background + layout + eyebrow so
   they don't look like seven copies of one image.
3. **Record the variant used** for each asset so the next one in the series can deliberately differ.
4. Light and dark must both appear in any multi-asset set unless the user pins one.

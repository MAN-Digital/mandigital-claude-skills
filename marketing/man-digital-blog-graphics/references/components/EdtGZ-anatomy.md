Source: `/Users/romeoman/Documents/Marketing/Design/Pencil/Playground.pen`

## Composite Role

Use `EdtGZ` when the graphic needs to explain a hybrid prospecting architecture: HubSpot for context and coordination, dedicated outbound tools for scale and deliverability, and a decision point that routes known high-intent accounts differently from cold scale motions.

## Routing Logic

- Use `qBrvo` for known accounts where HubSpot context and ownership matter.
- Use `L4Tph8` for cold or scaled outbound where inbox infrastructure and deliverability control matter.
- Use `tMycI`, `qctTA`, and `L4BsQ` together when the frame needs an explicit decision gate.
- Use `LcGj0` to make the strategic principle clear: split execution, connect context.
- Use `wTMqf` to close the architecture with the engagement-to-sales handoff.

## Reuse Rules

- Clone `EdtGZ` for the whole architecture.
- Clone `qBrvo` and `L4Tph8` as a paired lane system where possible.
- Clone `tMycI`, `qctTA`, `L4BsQ`, and connector nodes together if the routing logic remains.
- Keep connector paths blue and restrained.
- If the number of lane items changes, re-layout the card height and screenshot before export.

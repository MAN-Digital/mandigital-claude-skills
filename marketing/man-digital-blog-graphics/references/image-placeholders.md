# Image Placeholders

Some Playground templates include image or screenshot areas. Use them deliberately; do not fill them with generic stock-like decoration.

Image placeholders are the exception, not the build method. The final `.pen` design must remain editable. Do not export the whole design as a flat image and place it into a canvas or placeholder. See `editable-pencil-source.md`.

## Rules

- If the source node already has a screenshot/image placeholder, preserve the placeholder structure unless the user provides an approved image.
- If the graphic concept needs a realistic image but no approved image is available, leave a clearly named placeholder and tell the user what image belongs there.
- For generated imagery, create or request the image separately, then place it into the placeholder frame. Do not silently generate images inside production graphics unless the user asked for image generation.
- Use MAN Digital design-system assets first for logos, HubSpot badges, UI screenshots, and approved brand visuals.
- Do not use off-brand stock imagery for strategic RevOps graphics.
- Do not use a generated image, screenshot, or exported PNG as a replacement for editable Pencil notes, labels, cards, connectors, or diagrams.

## Useful Patterns

- `kRJNr` includes a 16:9 contrarian slide structure with a large visual/content area (`compF`) that can carry an image-like panel, screenshot, or diagram.
- `tMsEe` has integration image placeholders under the integration source card; replace only with approved logos/assets.
- Prospecting-Agent sequence frames with checkerboard screenshot carriers should stay placeholders until approved screenshots are available.

## User-Facing Note

When leaving a placeholder, explicitly say what belongs there, for example:

`Image placeholder left in the right panel: use an approved CRM screenshot or generated abstract signal-system image before publication.`

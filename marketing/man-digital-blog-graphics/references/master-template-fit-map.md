# Master Template Fit Map

Use this before choosing a layout for any HubSpot, RevOps, CRM, signal, routing, scoring, AI, revenue-engine, workflow, pipeline, or operating-model graphic.

These five Playground nodes are first-class systems, not optional examples:

- `nRPmP`
- `KVqAt`
- `tMsEe`
- `b8SoH`
- `llyux`

For open-ended MAN Digital blog graphics, evaluate all five before building. The result can be:

- `primary` - use the full node or its main structure as the base.
- `secondary` - borrow its card, connector, icon, label, or depth system.
- `depth` - borrow its decoration/background/brand treatment only.
- `not fit` - explain why another master or registry family fits better.

If none are used, the audit must say why the prompt is outside their coverage.

## Static Image / Mobile Reality

Pencil graphics export as static images. They do not reflow on mobile. A 1536px-wide graphic displayed in a 375px viewport is scaled to 24.4% of its design size. Exporting at 2x improves sharpness, but it does not make 20px source text readable.

Before building, decide the smallest expected display width:

- Blog body mobile: assume 343-375 px wide.
- Blog hero or OG preview: assume 375-600 px visible width depending on placement.
- LinkedIn feed mobile: assume the post is scanned quickly, often below full device width.
- Print/A4: mobile rules may not apply, but print-safe type still matters.

Use this rule:

- If text must be read on mobile, design it as big text in the source frame.
- If a detail cannot be made readable, treat it as texture or remove it.
- If the concept requires many readable details, choose a taller inline explainer or split the communication into multiple graphics when the prompt allows it.
- If the user asks for one single image, simplify the model instead of shrinking type.

```markdown
Master template scan:
- nRPmP: secondary/depth - use radar arcs and process strip, but full flywheel is too dense.
- KVqAt: primary - HubSpot data layer and stage-card architecture match the prompt.
- tMsEe: not fit - provider/source taxonomy is not the main story.
- b8SoH: secondary - central signal hub could support the trigger visual.
- llyux: not fit - implementation blueprint is broader than the prompt.
- Mobile plan: 1536px canvas displayed at 375px; key labels must be 42px+ source or converted to visual markers.
```

The final audit should be able to trace the output back to this decision.

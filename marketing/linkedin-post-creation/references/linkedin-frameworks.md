# LinkedIn Post Frameworks - 10 Proven Structures

Based on analysis from the LinkedIn Post Frameworks JSON file and 20 proven patterns.

## Framework Selection Guide

**Choose your framework based on content type:**

| Content Type | Recommended Framework | Why |
|--------------|----------------------|-----|
| Contrarian Opinion | Unpopular Opinion | Captures attention, sparks debate |
| Data/Statistics | Data-Driven Lead | Builds credibility with numbers |
| Methodology | Hook Framework System (Matt Swain) | Systematic, educational |
| Problem Solution | Use Case Framework (Fletch) | Clarifies value proposition |
| Personal Story | Story Format | Builds connection, relatability |
| List of Tools/Tips | Basic Listicle | Easy to scan, high shareability |
| Case Study | Case Study Format | Demonstrates results, proof |
| Industry Trends | Industry Observation | Positions as thought leader |
| How-To Guide | Step-by-Step Listicle | Actionable, practical value |
| Prediction | Prediction Format | Forward-looking, engaging |

## Framework 1: Unpopular Opinion

**Purpose:** Generate engagement through controversial opinions

**Best For:** Thought leaders, industry experts, established personal brands

**Structure:**
```
Line 1: Unpopular opinion: [Contrarian viewpoint]

Line 2: Everyone thinks [conventional approach], but here's the reality:

Lines 3-6:
• [Supporting evidence bullet 1]
• [Supporting evidence bullet 2]
• [Supporting evidence bullet 3]

Line 7: [Action] this. [Action] that instead.

Line 8: What's your [opinion/take] on [topic]?
```

**Example:**
```
Unpopular opinion: Most "productivity hacks" are procrastination in disguise.

Everyone's obsessed with finding the perfect system, but here's the truth:
• Spending 2 hours organizing your to-do list isn't productive
• Switching between 5 different apps creates more chaos
• The best system is the one you actually use consistently

Real productivity comes from doing the work, not optimizing the process.

What's your take? Am I completely off base here?
```

```
• [Implication 2]
• [Implication 3]
```

## Framework 3: Hook Framework System (Matt Swain)

**Purpose:** Systematic approach to creating attention-grabbing hooks

**Best For:** Educators, content creators, B2B professionals

**Structure:**
```
Line 1: I've [analyzed/reviewed] [large number]+ [items] that [result].

Line 2: Here's the [framework] I give [target audience]:

Lines 3-16: 7 steps with explanations

Line 17: Save this framework
```

**Example:**
```
I've analyzed 500+ LinkedIn hooks that generated 1M+ impressions.

Here's the hook framework I give B2B marketers:

1. Lead with transformation + numbers
   "From 100 to 10,000 customers in 12 months"

2. Show, don't tell your expertise
   "After building 50+ content systems..."

3. Reframe generic phrases
   Instead of "I often get asked..."
   Say: "The #1 question from Series B founders..."

4. Speak directly to your ICP
   "Here's why SaaS scaleups should listen..."

5. Package insights into frameworks
   "The 4-step Attribution System we use..."

6. Use names with existing attention
   "HubSpot scaled to $1B ARR. Here's their RevOps strategy..."

7. Make it feel exclusive
   "What I learned in $5000/hour consulting sessions..."

Save this for your next post.
```

## Framework 4: Use Case Framework (Anthony Pierri/Fletch)

**Purpose:** Articulate customer problems better than they can

**Best For:** B2B SaaS, product marketers, positioning work

**Structure:**
```
Line 1: If you want your [product/service] to [outcome]...

Line 2: USE CASE: What are [customers] trying to do?

Line 3: CURRENT WAY: How are they doing it now?

Line 4: LIMITATION: What's the constraint?

Line 5: PROBLEM: How does this hurt their [goal]?

Line 6: The solution? [Your approach].
```

**Example:**
```
If you want your HubSpot to scale with your SaaS company...

USE CASE: What are scaleups trying to do?
Track attribution across multiple channels and touchpoints to understand what's driving revenue.

CURRENT WAY: How are they doing it now?
Cobbling together spreadsheets, basic HubSpot reports, manual tracking across different tools.

LIMITATION: What's the constraint?
Manual processes break down at scale, data silos create blind spots.

PROBLEM: How does this hurt growth?
Revenue teams flying blind, marketing spend inefficient, can't identify which channels to double down on.

The solution? Advanced HubSpot architecture designed for scale.
```

**Note:** Remove colored square emojis for B2B professional tone!

**Structure:**

```
Line 3: Then we [implemented/discovered] our [method]:

Lines 4-7:
• [Solution step 1]
• [Solution step 2]
• [Solution step 3]

Line 8: The result after [timeframe]?

Lines 9-12:
• [Result 1]
• [Result 2]
• [Result 3]

Line 13: The lesson? [Key insight].
```

**Example:**
```
Two years ago, a SaaS scaleup came to us with $100K HubSpot investment that wasn't working.

They had all the tools but couldn't track what drove revenue.

Then we implemented our 4-pillar optimization framework:
• Cleaned up their data architecture
• Built proper attribution tracking
• Created executive dashboards that mattered
• Trained their team on advanced features

The result after 90 days?
• 300% improvement in lead attribution accuracy
• $2M in previously hidden pipeline identified
• 50% reduction in time spent on manual reporting
• ROI of 400% on their HubSpot investment

The lesson? It's not about having more tools. It's about using the right tools correctly.
```

## Framework 6: Basic Listicle

**Purpose:** Create scannable, shareable list content

**Best For:** All content creators, educational content

**Structure:**
```
Line 1: [Number] [topic] mistakes that are [negative outcome]:

Lines 2-15:
[Number]. [Mistake title]
[Brief explanation]

Line 16: Which one hits closest to home?

Line 17: Save this for your next [relevant situation]
```

**Example:**
```
7 HubSpot mistakes killing your revenue attribution:

1. Dirty contact data
Garbage in, garbage out. Clean your database first.

2. Missing UTM parameters
Can't track what you don't tag properly.

3. Broken form tracking
Forms without proper source attribution create blind spots.

4. Ignored offline touchpoints
Sales calls and events matter for attribution too.

5. Wrong attribution model
First-touch doesn't work for long B2B sales cycles.

6. Siloed data sources
Your CRM, marketing automation, sales tools need to talk.

7. No regular audits
Attribution accuracy degrades over time without maintenance.

Which one is hurting your revenue visibility right now?

Save this for your next HubSpot audit.
```

**Best For:** Agencies, consultants, service providers

**Structure:**
```
Line 1: Case Study: How a [company description] [achieved result]

Line 2: The Challenge:
Line 3: [Problem description]

Line 4: Our Approach:
Lines 5-8:
• [Solution step 1]
• [Solution step 2]
• [Solution step 3]

Line 9: The Results:
Lines 10-13:
• [Result 1]
• [Result 2]
• [Result 3]

Line 14: Key Takeaways for [audience]:
Line 15: [Main lesson]
```

**Example:**
```
Case Study: How a 25-person SaaS startup increased qualified leads by 300%

The Challenge:
Early-stage SaaS company using HubSpot like a basic CRM. No automation, manual lead qualification, zero visibility into what drove revenue.

Our Approach:
• Set up 5 core workflows for lead nurturing
• Implemented proper lead scoring based on product usage
• Created attribution tracking for outbound efforts
• Built simple dashboards for the founding team

The Results:
• 300% increase in qualified leads (10 to 40 per month)
• 50% reduction in time on manual tasks
• 85% improvement in lead-to-customer conversion
• 400% ROI within 90 days

Key Takeaways for early-stage founders:
You don't need complex systems. You need the RIGHT systems.
```

## Mobile-First Formatting Rules (Apply to ALL Frameworks)

**CRITICAL: These formatting rules apply to EVERY framework**

1. **One sentence per line** - Always
2. **Blank line between sentences** - Always
3. **No paragraphs** (2+ sentences together) - Never
4. **Short sentences** (10-15 words ideal) - As much as possible
5. **Bullets for lists** - Always
6. **NO EMOJIS** - Ever (B2B professional tone)
7. **Bold sparingly** - Maximum 1-2 times per post
8. **Mobile preview test** - Before finalizing

## Framework Selection Decision Tree

```
START: What's your primary goal?

├─ Generate engagement/debate
│   └─ Use: Unpopular Opinion

├─ Establish credibility with data
│   └─ Use: Data-Driven Lead

├─ Teach a methodology
│   └─ Use: Hook Framework System OR Use Case Framework

├─ Share results/proof
│   ├─ Personal story → Use: Story Format
│   ├─ Client work → Use: Case Study Format
│   └─ Transformation → Use: Transformation Format

├─ Provide tactical tips
│   └─ Use: Basic Listicle

├─ Comment on industry
│   ├─ Current trends → Use: Industry Observation
│   └─ Future trends → Use: Prediction Format

└─ Not sure?
    └─ Start with Basic Listicle (safest, highest shareability)
```

## Common Framework Mistakes

**Mistake 1: Using Too Many Frameworks in One Post**
- Stick to ONE framework per post
- Don't mix unpopular opinion with case study format
- Keep structure clean and consistent

**Mistake 2: Ignoring Mobile Formatting**
- Framework doesn't matter if formatting is wrong
- Always apply mobile-first rules
- Preview on phone before posting

**Mistake 3: Choosing Wrong Framework for Content**
- Don't force a story into a listicle format
- Match framework to your natural content type
- If it feels forced, try different framework

**Mistake 4: Copying Template Exactly**
- Frameworks are guides, not scripts
- Adapt to your voice and audience
- Make it authentic to you

**Mistake 5: No Hook**
- Every framework needs a strong opening
- First 2-3 lines must stop the scroll
- Apply hook principles even within framework

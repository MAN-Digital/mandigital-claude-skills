# 11-Point Editing Framework for LinkedIn Posts

## Overview

Apply these 11 editing checks to every LinkedIn post AFTER drafting and formatting. Each framework addresses a specific aspect of quality writing adapted for LinkedIn's mobile-first environment.

**Order Matters:** Apply frameworks 1-9 first (content), then 10 (grammar), then 11 (final polish).

---

## Framework 1: Redundancy

**Purpose:** Remove repeated words, phrases, and unnecessary filler

**What to Look For:**
- Repeated words within 2-3 sentences
- Filler words: "very", "really", "just", "that", "actually", "basically"
- Redundant phrases: "end result", "free gift", "past history"
- Repeated ideas stated differently

**LinkedIn-Specific:**
- Posts are short—every word must earn its place
- Mobile readers scan quickly—redundancy breaks flow
- Repetition signals lack of editing

**How to Fix:**

**Before:**
```
HubSpot attribution is very important for tracking what's actually driving your revenue. Really understanding attribution is critical for making smart decisions about where to invest.
```

**After:**
```
HubSpot attribution shows what drives revenue.

This clarity guides investment decisions.
```

**Common Redundancies:**

| Redundant | Better |
|-----------|--------|
| "very unique" | "unique" |
| "completely eliminate" | "eliminate" |
| "absolutely essential" | "essential" |
| "end result" | "result" |
| "past history" | "history" |
| "plan ahead" | "plan" |

**Editing Process:**
1. Read post aloud
2. Mark repeated words within 3-sentence proximity
3. Cut or replace with synonyms
4. Remove all filler words
5. Verify meaning remains clear

## Framework 2: Active Voice

**Purpose:** Convert passive constructions to active for stronger, clearer writing

**Why It Matters on LinkedIn:**
- Active voice is more engaging
- Passive voice sounds corporate and distant
- Mobile readers need direct, clear language
- Active voice builds stronger connection

**Passive vs Active:**

**Passive:** Subject receives the action
```
"The report was created by the team."
"Attribution was improved by cleaning the data."
"The decision was made to implement HubSpot."
```

**Active:** Subject does the action
```
"The team created the report."
"Cleaning the data improved attribution."
"We decided to implement HubSpot."
```

**How to Identify Passive Voice:**
- Look for "was/were/is/are + past participle"
- Check if you can add "by zombies" after the verb (if yes, it's passive)
- Subject is being acted upon (not acting)

**LinkedIn Examples:**

**Before (Passive):**
```
Attribution accuracy was increased by 300% after the data was cleaned by our team.
```

**After (Active):**
```
We cleaned the data.

Attribution accuracy increased 300%.
```

**Before (Passive):**
```
The HubSpot implementation was completed, and workflows were set up by the RevOps team.
```

**After (Active):**
```
The RevOps team completed the HubSpot implementation and set up workflows.
```

**When Passive Is OK:**
- When the actor is unknown or unimportant
- When emphasizing the action over the actor
- When you want to soften a statement

**Editing Process:**
1. Search for "was", "were", "is", "are" + past tense verb
2. Identify the real actor (who's doing the action?)
3. Rewrite with actor as subject
4. Verify sentence is stronger and clearer

## Framework 3: Parallelism

**Purpose:** Create consistent structure in lists, bullets, and related sentences

**Why It Matters on LinkedIn:**
- Parallelism creates rhythm and flow
- Consistent structure is easier to scan
- Professional polish signals attention to detail
- Particularly important in bullet lists

**Parallel Structure Rules:**

1. **Start bullets the same way**
```
❌ Not Parallel:
• Cleaning your data
• Set up attribution
• You should train your team

✅ Parallel:
• Clean your data
• Set up attribution
• Train your team
```

2. **Match grammatical forms**
```
❌ Not Parallel:
• To improve attribution
• Cleaning data helps
• You should implement workflows

✅ Parallel (all infinitives):
• To improve attribution
• To clean data
• To implement workflows

OR (all imperatives):
• Improve attribution
• Clean data
• Implement workflows
```

3. **Maintain consistent tense**
```
❌ Not Parallel:
• We cleaned the data
• Setting up workflows
• Will train the team

✅ Parallel (all past tense):
• We cleaned the data
• We set up workflows
• We trained the team
```

**LinkedIn Examples:**

**Before (Not Parallel):**
```
The companies getting ROI typically:
• Have clean data
• Proper attribution tracking
• They train their teams
• Auditing regularly
```

**After (Parallel):**
```
The companies getting ROI typically:
• Maintain clean data
• Track attribution properly
• Train their teams
• Audit regularly
```

**Editing Process:**
1. Identify all bullet lists
2. Check first word of each bullet
3. Ensure same grammatical form
4. Apply same tense throughout
5. Verify rhythm and scannability

## Framework 4: Tenses

**Purpose:** Maintain consistent verb tenses and fix inappropriate shifts

**Why It Matters on LinkedIn:**
- Tense shifts confuse readers
- Consistency maintains clarity
- Professional writing stays in appropriate tense
- Errors break trust

**Common Issues:**

**Issue 1: Unnecessary Past-Present Shifts**
```
❌ Inconsistent:
I audited 200+ instances. I see the same pattern. Most companies were making this mistake.

✅ Consistent:
I've audited 200+ instances. I see the same pattern. Most companies make this mistake.
```

**Issue 2: Story Tense Confusion**
```
❌ Confusing:
A client came to us last year. They have $100K in HubSpot spend. We implement our framework.

✅ Clear:
A client came to us last year. They had $100K in HubSpot spend. We implemented our framework.
```

**LinkedIn Tense Guidelines:**

**Use Present Tense For:**
- Current truths: "HubSpot attribution matters"
- Ongoing situations: "Most companies struggle with data"
- General advice: "Clean your data first"

**Use Past Tense For:**
- Specific past events: "We cleaned their data"
- Case studies: "They increased attribution 300%"
- Personal stories: "I learned this lesson"

**Use Present Perfect For:**
- Experience spanning to now: "I've audited 200+ instances"
- Ongoing relevance: "We've seen this pattern repeatedly"

**Editing Process:**
1. Identify the main timeframe of your post
2. Check each verb for tense
3. Mark inappropriate shifts
4. Rewrite for consistency
5. Verify clarity improved

## Framework 5: Sentence Structure

**Purpose:** Vary sentence length and structure while maintaining brevity

**Why It Matters on LinkedIn:**
- Variety prevents monotony
- Short sentences create urgency
- Strategic longer sentences add context
- Mix creates rhythm

**LinkedIn Sentence Guidelines:**

**Ideal Mix:**
- 70% short sentences (5-10 words)
- 20% medium sentences (11-15 words)
- 10% longer sentences (16-20 words)
- Avoid 20+ word sentences

**Structure Patterns:**

**Pattern 1: Short Impact**
```
Clean your data.

Set up attribution.

Train your team.

That's the foundation.
```

**Pattern 2: Short + Medium Mix**
```
HubSpot attribution clarity matters.

Without it, you're flying blind on revenue.

Every marketing dollar becomes a guess.
```

**Pattern 3: Occasional Longer for Context**
```
After auditing 200+ HubSpot implementations across SaaS companies at different growth stages, I've identified one consistent pattern.

Most companies implement features before cleaning data.

It's backwards.
```

**Before:**
```
Most SaaS companies think that more HubSpot features will equal better results, but I've audited over 200 instances and the pattern is clear that complex workflows that are built on dirty data break more often than they help, which is why the companies that get ROI do one thing first.
```

**After:**
```
Most SaaS companies think more HubSpot features = better results.

But I've audited 200+ instances.

The pattern is clear: Complex workflows built on dirty data break more than they help.

Companies getting ROI do one thing first.
```

**Editing Process:**
1. Count words in each sentence
2. Look for sentences over 20 words
3. Break long sentences into multiple short ones
4. Verify mix creates rhythm
5. Read aloud to test flow

## Framework 6: Specificity

**Purpose:** Replace generic language with specific, concrete details

**Why It Matters on LinkedIn:**
- Specific = credible
- Generic = forgettable
- Numbers and names build trust
- Precision captures attention

**Generic to Specific Transformations:**

**Numbers:**
```
❌ Generic: "Many companies"
✅ Specific: "200+ companies"

❌ Generic: "Recent research"
✅ Specific: "2024 Gartner study"

❌ Generic: "Significant improvement"
✅ Specific: "300% improvement"
```

**Outcomes:**
```
❌ Generic: "Better results"
✅ Specific: "Cut 15 hours of manual reporting weekly"

❌ Generic: "Improved attribution"
✅ Specific: "Attribution accuracy jumped from 10% to 85%"
```

**Actions:**
```
❌ Generic: "Optimize your system"
✅ Specific: "Clean contact data, set up lead scoring, audit monthly"

❌ Generic: "Implement best practices"
✅ Specific: "Use first-touch attribution for awareness, last-touch for conversion"
```

**LinkedIn Examples:**

**Before:**
```
We helped a company improve their HubSpot setup. They saw better results.
```

**After:**
```
We audited a 25-person SaaS startup's HubSpot implementation.

Within 90 days: 300% more qualified leads, 50% less manual work, 400% ROI.
```

**Editing Process:**
1. Highlight vague words: "many", "some", "better", "improved"
2. Replace with specific numbers
3. Quantify outcomes when possible
4. Add names, dates, details
5. Verify increased credibility

## Framework 7: Amplify Arguments

**Purpose:** Strengthen weak claims with evidence, specificity, and impact

**Why It Matters on LinkedIn:**
- Weak claims get scrolled past
- Strong arguments stop the scroll
- Evidence builds credibility
- Impact creates urgency

**How to Amplify:**

**1. Add Evidence**
```
❌ Weak: "Data quality matters"
✅ Amplified: "73% of companies make decisions on unreliable data (2024 RevOps Report)"
```

**2. Add Consequences**
```
❌ Weak: "Clean your data"
✅ Amplified: "Clean your data or waste $50K+ on automation that breaks"
```

**3. Add Specificity**
```
❌ Weak: "Attribution helps"
✅ Amplified: "Attribution accuracy jumped from 10% to 85% after implementing these 3 changes"
```

**4. Add Contrast**
```
❌ Weak: "This approach works"
✅ Amplified: "Most companies implement features first. We do the opposite—and see 300% better results."
```

**LinkedIn Examples:**

**Before:**
```
HubSpot can help your company.

You should use it properly.

The results will be good.
```

**After:**
```
HubSpot can transform revenue operations.

But 73% of companies underutilize their $50K+ annual investment.

The difference? Clean data foundation before automation.

Companies that do this see 400% ROI. Companies that don't waste their budget on broken workflows.
```

**Editing Process:**
1. Identify claims without support
2. Add data, examples, or consequences
3. Increase specificity
4. Create contrast or comparison
5. Verify increased impact

## Framework 8: Structural Strategy

**Purpose:** Verify logical flow and effective progression of ideas

**Why It Matters on LinkedIn:**
- Posts must flow naturally
- Each sentence should lead to the next
- Hook must deliver on promise
- CTA should feel natural

**Flow Checklist:**

**1. Hook Delivers**
```
Hook promises: "The #1 HubSpot mistake"
Body must: Actually reveal the #1 mistake clearly
```

**2. Logical Progression**
```
✅ Good Flow:
Problem → Consequence → Solution → Result

❌ Broken Flow:
Problem → Solution → Problem Again → Result
```

**3. Transitions Work**
```
❌ Abrupt: "Data quality matters. Set up workflows."
✅ Smooth: "Data quality matters. Once clean, you can build reliable workflows."
```

**4. CTA Matches Content**
```
If content is about attribution:
❌ Wrong CTA: "What's your favorite HubSpot feature?"
✅ Right CTA: "What's your biggest attribution challenge?"
```

**Editing Process:**
1. Read post top to bottom
2. Check if hook promise is fulfilled
3. Verify each sentence connects to next
4. Ensure conclusion follows from body
5. Confirm CTA relates to content

## Framework 9: Takeaway

**Purpose:** Ensure the key message is crystal clear

**Why It Matters on LinkedIn:**
- Readers must leave with ONE clear idea
- If they forget everything else, what should they remember?
- LinkedIn posts need memorable conclusions

**Takeaway Rules:**

**1. One Main Point**
```
❌ Too Many: "Clean data, set up attribution, train teams, audit monthly, use automation"
✅ Clear: "Clean data before building automation"
```

**2. Memorable Phrasing**
```
❌ Forgettable: "You should do data quality work first"
✅ Memorable: "You're building on quicksand if your data is dirty"
```

**3. Action-Oriented**
```
❌ Passive: "Data quality is important"
✅ Active: "Clean your data before your next HubSpot project"
```

**LinkedIn Examples:**

**Before:**
```
So yeah, attribution is pretty important and you should probably think about your data quality and maybe implement some workflows or something.
```

**After:**
```
The takeaway: Features don't fix dirty data.

Clean your foundation first. Build automation second.

That's how you turn a $50K HubSpot investment into 400% ROI.
```

**Editing Process:**
1. Identify your ONE main message
2. State it clearly in conclusion
3. Make it memorable
4. Ensure it's actionable
5. Verify it matches the hook

## Framework 10: Grammar & Spelling

**Purpose:** Fix all mechanical errors

**Why It Matters on LinkedIn:**
- Errors break trust
- Professional posts are error-free
- Grammar mistakes signal carelessness
- Credibility depends on polish

**Common LinkedIn Errors:**

**1. Wrong "Your/You're"**
```
❌ "Your going to waste budget"
✅ "You're going to waste budget"
```

**2. Wrong "Its/It's"**
```
❌ "HubSpot and it's features"
✅ "HubSpot and its features"
```

**3. Missing Commas**
```
❌ "After cleaning data set up workflows"
✅ "After cleaning data, set up workflows"
```

**4. Incorrect Apostrophes**
```
❌ "The company's improved their attribution"
✅ "The companies improved their attribution"
```

**5. Common Typos**
- "manger" → "manager"
- "form" → "from"
- "lead" → "leads" (singular/plural)

**Editing Process:**
1. Read slowly, word by word
2. Check each sentence structure
3. Verify subject-verb agreement
4. Fix punctuation
5. Run through spell checker
6. Read aloud to catch errors

## Framework 11: Final Polish

**Purpose:** Word-by-word refinement for maximum impact

**Why It Matters on LinkedIn:**
- Final pass catches what others missed
- Micro-improvements compound
- Polish separates good from great
- This is your last chance to improve

**Polish Checklist:**

**1. Wording Improvements**
```
❌ Weak: "We made things better"
✅ Strong: "We increased attribution accuracy 300%"
```

**2. Brevity Check**
```
❌ Wordy: "In order to improve your results"
✅ Brief: "To improve results"

❌ Wordy: "Due to the fact that"
✅ Brief: "Because"
```

**3. Remove Hedging**
```
❌ Uncertain: "I think maybe you should probably consider"
✅ Confident: "You should"

❌ Uncertain: "It seems like this might help"
✅ Confident: "This helps"
```

**4. Strengthen Weak Verbs**
```
❌ Weak: "is", "has", "get", "make", "do"
✅ Strong: "drives", "generates", "transforms", "achieves"
```

**5. Mobile Preview (CRITICAL)**
- View on phone screen
- Check line breaks
- Verify scannability
- Ensure no sentences too long
- Confirm NO EMOJIS

**6. Read Aloud**
- Does it sound natural?
- Are there tongue-twisters?
- Does rhythm work?
- Is flow smooth?

**7. Check Against Mobile Rules:**
- [ ] One sentence per line
- [ ] Spaces between sentences
- [ ] No paragraphs
- [ ] Short sentences
- [ ] Bullets properly formatted
- [ ] NO EMOJIS
- [ ] Bold used sparingly

**8. Verify Hook Quality**
- [ ] Specific
- [ ] Contrarian or unexpected
- [ ] Emotional
- [ ] Relatable
- [ ] Credibility marker
- [ ] Cliffhanger

**9. Check Voice**
- Sounds like you (not robotic)
- Confident (not hedging)
- Professional (not casual)
- Direct (not verbose)

**Editing Process:**
1. Read entire post slowly
2. Mark any weak words
3. Strengthen verbs
4. Remove hedging
5. Preview on mobile
6. Read aloud
7. Check all mobile rules
8. Verify hook quality
9. Confirm authentic voice

## Editing Workflow Summary

**Phase 1: Content (Frameworks 1-9)**
1. Redundancy - Cut repeated words and filler
2. Active Voice - Convert passive to active
3. Parallelism - Match structures
4. Tenses - Maintain consistency
5. Sentence Structure - Vary length, keep short
6. Specificity - Add concrete details
7. Amplify - Strengthen arguments
8. Structure - Verify logical flow
9. Takeaway - Clarify main message

**Phase 2: Mechanics (Framework 10)**
10. Grammar & Spelling - Fix all errors

**Phase 3: Polish (Framework 11)**
11. Final Polish - Word-by-word refinement

## Before & After Example

**Before (No Editing):**
```
I think that maybe HubSpot attribution is something that could be important for your business and you should probably think about implementing it because it might help you understand where your leads are actually coming from and that information could be useful for making decisions about your marketing budget allocation which is something that many companies struggle with these days and I've seen this pattern across a lot of different companies that I've worked with.
```

**After (All 11 Frameworks Applied):**
```
HubSpot attribution reveals what drives revenue.

Without it, you're flying blind on $50K+ marketing spend.

After auditing 200+ implementations, I've found the #1 blocker: dirty data.

Companies fix this first see 4x ROI.

Companies that don't waste their budget.

Clean your data before your next campaign.

What's your biggest attribution challenge?
```

**What Changed:**
1. Redundancy - Removed filler ("I think", "maybe", "probably", "could be")
2. Active Voice - Changed passive constructions
3. Parallelism - N/A (no lists in this example)
4. Tenses - Made consistent (present tense)
5. Sentence Structure - Broke into short, punchy sentences
6. Specificity - Added "200+", "$50K+", "4x ROI", "auditing"
7. Amplify - Added consequences, contrast, evidence
8. Structure - Created logical flow: problem → insight → solution
9. Takeaway - Clear main message and CTA
10. Grammar - Fixed all errors
11. Polish - Mobile formatting, confident voice, strong verbs
